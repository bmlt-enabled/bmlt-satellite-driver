1 DESCRIPTION
2 -----------
3 
4 This is a class that is to be used by PHP coders, for writing custom satellite implementations
5 of the BMLT (Basic Meeting List Toolbox).
6 
7 It is a class that handles the controller layer in the BMLT client/server system. This class will
8 deal with all HTTP interactions, and uses the XML interface of the BMLT root server to operate.
9 
10 It is designed to provide a functional interface to a communication infrastructure for
11 BMLT satellites. It communicates with the root server using the XML variant of the REST
12 interface.
13 
14 This class will not use a class to hold transactions or parameters. This is because [the
15 fox ain't worth the chase](http://philip.greenspun.com/humor/eecs-difference-explained)
16 That being said, transactions can be serialized and restored. This class is to be used as
17 a "driver," not a communication stack. It is up to the implementation to do things like
18 manage multiple transactions and whatnot.
19 
20 REQUIREMENTS
21 ------------
22 
23 This class requires a root server version of at least 1.8.1, and 1.8.31 introduces some new details
24 that will add more capability. It *MAY* work with older root servers, but there are no guarantees as
25 to the level of functionality available.

32 bmlt_satellite_controller.

35
